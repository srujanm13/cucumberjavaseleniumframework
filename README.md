# Cucumber Selenium Test Framework

This project is a Java 11 Selenium + Cucumber + TestNG automation framework built with Maven. It supports local and remote execution, TestNG suite-based orchestration, environment selection through Maven profiles, JSON-driven negative test data, Extent reporting, and Log4j2 scenario/execution logs.

## Tech Stack

- Java 11
- Maven
- Selenium 4
- Cucumber 7
- TestNG 7
- Extent Reports 5
- Log4j2

## Prerequisites

- JDK 11 installed and `JAVA_HOME` configured
- Maven installed and available in `PATH`
- At least one supported browser installed locally: Chrome, Firefox, or Edge
- Internet access to reach `https://rahulshettyacademy.com/loginpagePractise/`
- Selenium Grid available at the configured `gridUrl` when running with remote execution

## How Test Execution Works

1. Maven Surefire reads `suiteXmlFile` from `pom.xml`. By default it points to `suites/suite.xml`.
2. TestNG starts the configured runner classes from the suite XML.
3. `BaseCucumberTestNGRunner` copies suite parameters like `browser`, `device`, `env`, `execution`, and `gridUrl` into `ExecutionContext`.
4. Cucumber hooks start the driver, report, and logs before each scenario.
5. Step definitions call page objects and utility methods.
6. On completion, reports are flushed and driver/logging resources are cleaned up.

## Ways To Execute Test Cases

### 1. Run the default suite with `mvn test`

This uses the default property `suiteXmlFile=suites/suite.xml`.

```bash
mvn test
```

### 2. Run a specific TestNG suite XML file

Run the default regression suite explicitly:

```bash
mvn test -DsuiteXmlFile=suites/suite.xml
```

Run the browser/device matrix suite:

```bash
mvn test -DsuiteXmlFile=suites/browser-device-suite.xml
```

### 3. Run a single feature file

Use the Cucumber system property to restrict execution to one feature file:

```bash
mvn test -Dcucumber.features=src/test/resources/features/Login.feature
```

You can also target a specific scenario line inside the feature file:

```bash
mvn test -Dcucumber.features=src/test/resources/features/Login.feature:13
```

### 4. Run by tag

Run only smoke scenarios:

```bash
mvn test -Dcucumber.filter.tags="@smoke"
```

Run only negative scenarios:

```bash
mvn test -Dcucumber.filter.tags="@negative"
```

Run combinations:

```bash
mvn test -Dcucumber.filter.tags="@smoke or @negative"
```

### 5. Run by Maven profile

The project defines three Maven profiles that set the `env` property.

Run against QA:

```bash
mvn test -Pqa
```

Run against Stage:

```bash
mvn test -Pstage
```

Run against Prod:

```bash
mvn test -Pprod
```

Profiles can be combined with other runtime overrides:

```bash
mvn test -Pqa -Dbrowser=firefox -Ddevice=mobile
```

### 6. Run with browser, device, and execution overrides

Local Chrome desktop:

```bash
mvn test -Dbrowser=chrome -Ddevice=desktop -Dexecution=local
```

Local Firefox mobile viewport:

```bash
mvn test -Dbrowser=firefox -Ddevice=mobile -Dexecution=local
```

Remote execution on Selenium Grid:

```bash
mvn test -Dexecution=remote -DgridUrl=http://localhost:4444/wd/hub -Dbrowser=chrome
```

## Maven Command-Line Arguments Supported By This Project

### Project-specific properties

| Argument | Example | Purpose |
| --- | --- | --- |
| `-DsuiteXmlFile=...` | `-DsuiteXmlFile=suites/browser-device-suite.xml` | Chooses which TestNG suite file Surefire executes. |
| `-Dbrowser=...` | `-Dbrowser=edge` | Selects browser for local execution or report metadata. Supported in code: `chrome`, `firefox`, `edge`. |
| `-Ddevice=...` | `-Ddevice=mobile` | Applies viewport sizing. Supported in code: `desktop`, `mobile`, `tab`/`tablet`. |
| `-Dos=...` | `-Dos=windows` | Passed into Surefire system properties. Mainly used as metadata/config context. |
| `-Denv=...` | `-Denv=qa` | Selects environment and environment-specific base URL lookup. |
| `-Dexecution=...` | `-Dexecution=remote` | Chooses local or remote driver creation. |
| `-DgridUrl=...` | `-DgridUrl=http://localhost:4444/wd/hub` | Selenium Grid hub URL used when `execution=remote`. |
| `-Dcucumber.features=...` | `-Dcucumber.features=src/test/resources/features/Login.feature` | Restricts execution to a feature file or feature line. |
| `-Dcucumber.filter.tags=...` | `-Dcucumber.filter.tags="@smoke"` | Filters scenarios by Cucumber tags. |
| `-Dcucumber.glue=...` | `-Dcucumber.glue=stepdefinitions,hooks` | Overrides glue package scanning if needed. |

| Argument | Example | Purpose |
| --- | --- | --- |
| `-P...` | `-Pqa` | Activates a Maven profile. |

## Reports And Logs

- Extent report: `reports/extent-report.html`
- Surefire/TestNG reports: `target/surefire-reports/`
- Execution logs: `logs/*.log`
- Scenario logs: `logs/scenarios/*.log`

The Maven clean plugin is configured to clean `logs/` and `reports/` during `mvn clean`.

## Configuration Notes

- Default browser in `pom.xml` is `chrome`.
- Default device in `pom.xml` is `desktop`, but `config.properties` currently sets `device=mobile`. Because `ConfigReader` prefers JVM system properties first, a `-Ddevice=...` override will win at runtime.
- Default environment in `pom.xml` is `qa`.
- `ConfigReader.getBaseUrl()` first tries `<env>.baseUrl` and then falls back to `baseUrl`.
- Remote execution currently creates only `ChromeOptions` in `RemoteDriverManager`, even if another browser is passed. If true cross-browser remote execution is needed, that class should be extended.

## Known Project Behavior

- The smoke step `Then verify login screen is displayed` currently contains `Assert.assertTrue(false)` in `src/test/java/stepdefinitions/LoginSteps.java`, so the smoke scenario is expected to fail until that step is implemented.
- The negative login scenario reads credentials and expected error messages from `src/main/resources/testdata/login.json`.
- Parallel execution is enabled in both Surefire and the Cucumber TestNG data providers.

## Suggested Onboarding Flow

1. Review `pom.xml` for default execution properties.
2. Review `suites/suite.xml` to see which runners are executed by default.
3. Review `src/test/resources/features/Login.feature` to understand the available scenarios and tags.
4. Review `src/test/java/stepdefinitions/LoginSteps.java` and `src/main/java/pages/LoginPage.java` to understand the feature-to-page flow.
5. Run `mvn clean test` and inspect `reports/extent-report.html` plus `target/surefire-reports/`.
